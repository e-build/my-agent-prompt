## 문서 목적

- 프로젝트 디자인 시스템의 진입점
- 각 문서의 역할과 읽는 순서 안내
- 디자인 일관성을 유지하기 위한 기준서

## 디자인 시스템을 보는 큰 구조

| 계층 | 설명 | 담당 문서 |
|------|------|----------|
| 원칙 | 디자인의 근본 규칙, 이유 | `principles.md` |
| 토큰 | 시각의 원자 단위 (색상/타이포/간격) | `tokens.md` |
| 컴포넌트 | 토큰을 조합한 UI 요소 | `components.md` |
| 패턴 | 컴포넌트를 조합한 화면 단위 구성 | `patterns.md` |

## 문서별 역할

- `principles.md` — 이 디자인 시스템이 왜 이렇게 생겼는지 설명
- `tokens.md` — 색상, 타이포그래피, 간격, 그림자 등 시각 원자값
- `components.md` — 버튼, 입력폼, 카드, 모달 등 UI 요소의 상태와 변형
- `patterns.md` — 화면 레이아웃, 데이터 표시, 에러/로딩/빈 상태 패턴

## 처음 읽는 추천 순서

1. `principles.md` — 왜 (가장 먼저)
2. `tokens.md` — 무엇 (두 번째)
3. `components.md` — 어떻게 (세 번째)
4. `patterns.md` — 언제 (네 번째)

## 관련 문서
