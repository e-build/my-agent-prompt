## 문서 목적

- 컴포넌트를 조합한 화면 단위 구성 패턴 정의
- 자주 반복되는 UI 구성을 표준화
- 일관된 사용자 경험 유지 목적

## 화면 레이아웃 패턴

### List / Detail

```
[목록 화면] → [상세 화면] → (액션)
```

### Form / Wizard

```
[입력 폼] → [검증] → [확인/결과]
```

## 데이터 표시 패턴

### 리스트

- Flat list: 단순 나열
- Grouped list: 카테고리별 그룹
- Card grid: 카드 형태 격자

### Empty 상태

데이터가 없을 때 표시할 UI 구성:

| 요소 | 설명 |
|------|------|
| 아이콘 | 빈 상태를 나타내는 일러스트 |
| 메시지 | "아직 데이터가 없습니다" |
| CTA | 데이터 추가 액션 |

## 상태 패턴

### Loading

데이터 로딩 중:
- Skeleton: 콘텐츠 형태를 유지한 스켈레톤
- Spinner: 중앙 또는 인라인 스피너

### Error

오류 발생 시:
- 인라인 오류: 입력값 오류
- Toast: 일회성 알림
- Full screen: 치명적 오류

### Success

성공 완료 후:
- Toast 메시지
- 확인 화면 전환
- 자동 이동

## 내비게이션 패턴

- Tab: 동일 레벨 섹션 전환
- Push: 상세 → 뒤로가기
- Modal: 독립 작업
- Bottom sheet: 간단한 선택/입력
