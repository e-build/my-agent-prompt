## 문서 목적

- 기능 문서 전반에 공통 적용하는 핵심 원칙 정리
- 가독성, 구조, 순서 일관성 확보 목적
- SKILL.md 문서 작성 규칙 요약의 상세화

## 구조 우선

- 긴 줄글 지양
- **글머리 기호**와 번호 매기기 적극 사용
- 비교, 절차, 기준, 예외는 목록으로 분리
- 문서 순서는 이해하기 쉬운 흐름 기준으로 설계

권장 흐름 예:

1. 배경 / 목적
2. 핵심 요약
3. 구조 / 규칙 / 절차
4. 예외 / 주의사항
5. 예시 / 참고자료

## 제목과 강조

- **H1, H2, H3** 위계 명확화
- 제목만 읽어도 흐름이 보이도록 구성
- 핵심 키워드, 수치, 결론은 **볼드체** 사용
- 모든 본문형 Markdown 문서에는 **H1 바로 아래 목차** 포함

## 문체 통일

- `~하다`, `~한다` 중심 서술 지양
- `~함`, `~필요`, `~목적`, `~기준`, `~상태` 같은 명사형 마무리 우선
- 설명보다 정리, 선언, 기준 제시에 맞는 문체 유지
- 문체는 기준형 / 명사형 우선

## 다이어그램 절제

- Mermaid 등 다이어그램은 **이해에 실제 도움이 될 때만** 사용
- 우선 사용 대상
  - 상태 전이
  - 시간 순서 흐름
  - 시퀀스
  - 복잡한 결정 흐름
- 단순 목록이나 짧은 설명을 억지로 다이어그램화 금지

## 파일명 규칙

- 여러 문서를 만들면 파일명 규칙 통일
- 문서명에 **날짜 포함 금지**
- 가능한 **소문자 + 하이픈** 형태 사용
  - 예: `service-overview.md`
- 같은 묶음 문서에서 파일명 규칙 혼용 금지

## 중복 정의 금지

- 같은 내용을 여러 문서에 중복 정의하지 않음
- 공통 기준은 한 곳(공통 문서)에 두고 기능 문서에서는 참조
- 기능 고유 내용만 각 기능 문서에 작성

## 금지 규칙

- 이모티콘 사용 금지
- 날짜 포함 파일명 금지
- 같은 묶음 문서에서 파일명 규칙 혼용 금지
- 문서-코드 불일치 상태로 방치 금지 (차이 발생 시 즉시 정정)
