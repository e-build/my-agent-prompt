## 문서 목적

- 프로젝트의 모든 화면 ID를 통합 관리하기 위한 레지스트리
- 기능별 `03-screens.md`와 연결되는 루트 참조 문서
- 화면 ID 중복 방지 및 일관성 확보 목적

## 화면 ID 체계

- 형식: `S-{도메인}-{화면명}`
- 예시:
  - `S-AUTH-LOGIN` — 로그인 화면
  - `S-ORDER-LIST` — 주문 목록
  - `S-PRODUCT-DETAIL` — 상품 상세

### ID 부여 규칙

- 도메인 축약어 사용 (예: AUTH, ORDER, PRODUCT, USER)
- 화면명은 대문자 + 하이픈 (예: LOGIN, LIST, DETAIL, CREATE)
- 하나의 기능에서 여러 화면이면 기능 축약어 공유
- 새 화면 추가 시 이 문서에 등록 후 `03-screens.md`에 상세 정의

## 전체 화면 목록

| 화면 ID | 기능 | 설명 | 담당 | 상태 |
|---|---|---|---|---|
| `S-{DOMAIN}-{SCREEN}` | 기능명 | 한 줄 설명 | 백엔드/모바일/웹 | 계획/작성중/완료 |

## 화면별 상세

### 1. `S-{DOMAIN}-{SCREEN}`

- **소속 기능**: 기능명 (F-XX)
- **사용자 목적**: 이 화면에서 사용자가 하는 일
- **진입 조건**: 이전 화면 / 액션
- **Primary CTA**: 대표 액션
- **Secondary CTA**: 보조 액션
- **상태 분기**:
  - 기본: 정상 상태 UI
  - Empty: 데이터 없음
  - Loading: 로딩 중
  - Error: 오류 발생
  - Success: 성공 후
- **정의 문서**: `docs/features/{feature}/v{version}/03-screens.md`
