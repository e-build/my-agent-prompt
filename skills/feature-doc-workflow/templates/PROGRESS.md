## 문서 목적

- 프로젝트의 **기능별 작업 목록과 현재 현황을 한눈에 확인**하기 위한 루트 대시보드
- 새 세션 시작 시 현재 위치, 주요 작업, 블로커, 상세 진행 문서 위치를 빠르게 파악
- 세부 체크리스트는 `docs/features/{feature}/v{version}/07-progress.md`에서 관리

## 작업 절차 기준

모든 기능 작업은 `AGENTS.md`의 **6단계 표준 절차**를 따른다.

| 단계 | 이름 | 산출물 위치 |
|---|---|---|
| 1 | 요구사항 정의 | `docs/features/{feature}/v{version}/01-requirements.md` |
| 2 | 기획 및 정책 수립 | `docs/features/{feature}/v{version}/02-plans.md` |
| 3 | 화면정의서 작성 | `docs/features/{feature}/v{version}/03-screens.md` |
| 4 | API 명세서 정의 (에러 코드 포함) | `docs/features/{feature}/v{version}/04-api.md` |
| 5 | 시스템 설계 | `docs/features/{feature}/v{version}/05-design.md` |
| 6 | 구현 — **분야별 분리 추적** | 백엔드 / 모바일 / 웹 등 |

- 단계 6은 분야별 진행이 다를 수 있으므로 하위 구분을 별도로 추적한다 (예: 6a 백엔드, 6b 모바일, 6c 웹).
- 이전 단계 합의 없이 다음 단계로 건너뛰지 않는다.

## 업데이트 규칙

- `PROGRESS.md`는 전체 현황판으로 유지하고 세부 체크리스트를 길게 넣지 않음
- 기능 진행 상태는 **기능별 6단계 진행 매트릭스** 기준으로 추적
- 기능별 작업 상태는 `docs/features/{feature}/v{version}/07-progress.md`에서 관리
- 세션 종료 전 `PROGRESS.md`와 관련 분야 문서를 함께 갱신
