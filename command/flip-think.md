# flip-think

뻔한 답만 떠오를 때 사용합니다.
설계, 개발, 비즈니스 등 어떤 도메인이든 적용 가능합니다.

## 사용 방법

```
/flip-think $PROBLEM
```

- `$PROBLEM`: 막혀 있거나 뻔한 답만 나오는 문제 상황을 자유롭게 서술한다.
- 생략 시 "어떤 문제 상황인지 설명해주세요"라고 먼저 물어봅니다.

---

## Phase 1 — 가정 발굴

문제 영역에서 사람들이 "당연히 그래야 한다"고 믿는 가정을 전수 열거한다.

- 가정 하나당 한 줄: **"[주체]는 [대상]이 [성질]이어야 한다고 믿는다"**
- 크고 작은 가정 모두 포함한다. 빠뜨리는 게 더 위험하다.

## Phase 2 — 가정 분류

각 가정을 두 유형으로 나눈다:

| 유형 | 정의 |
|------|------|
| **물리적 한계** | 자연 법칙·수학적 제약으로 실제로 불가능한 것 |
| **관성적 관습** | 과거 방식을 답습하거나 남들이 하니까 따라하는 것 |

> 관성적 관습은 깰 수 있다. 물리적 한계만이 진짜 벽이다.

## Phase 3 — 본질 추출

관성적 관습을 전부 제거했을 때 이 문제의 진짜 목표는 무엇인가?

- **한 문장**으로 본질을 정의한다.
- "기존 방식 없이 이 본질만 달성하려면?" 을 출발점으로 삼는다.

## Phase 4 — 역방향 구조 설계

본질에서 출발해 관성적 관습을 의도적으로 뒤집는 구조를 설계한다.

- 관습마다 "기존: [A] → 역방향: [not-A]" 형태로 명시
- 역방향이 실제로 작동 가능한지 물리적 한계 기준으로 검증
- 작동 가능한 역방향 조합으로 새 구조를 제안

## 출력 형식

```
## 발굴된 가정 목록
1. "..."
2. "..."
...

## 가정 분류
| 가정 | 유형 | 이유 |
|------|------|------|

## 본질 (First Principle)
> [한 문장]

## 역방향 구조 설계
- 기존: ... → 역방향: ...
- 기존: ... → 역방향: ...

## 제안 구조
[본질 + 역방향 조합으로 설계한 새로운 접근 방식]
```
