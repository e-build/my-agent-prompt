# flip-action-plan

실행 방향은 잡혔는데 어디서부터 시작해야 할지 막막할 때,
또는 과거에 비슷한 시도에서 실패한 패턴을 되풀이하고 싶지 않을 때 사용합니다.

설계, 개발, 비즈니스 등 어떤 도메인이든 적용 가능합니다.

## 사용 방법

```
/flip-action-plan $SOLUTION
```

- `$SOLUTION`: 실행하려는 해결책 또는 전략을 서술한다.
- `/flip-think` 결과를 그대로 붙여넣어도 됩니다.
- 생략 시 "어떤 해결책을 실행하려 하시나요?"라고 먼저 물어봅니다.

---

## Phase 1 — 액션 분해

해결책을 실행 가능한 최소 단위로 분해한다.

- 각 스텝은 **동사 + 목적어** 형태: "XX를 한다"
- 각 스텝마다 **완료 기준(Done Criteria)** 명시 — 완료 여부를 스스로 판단할 수 있어야 한다
- 병렬 가능한 스텝과 순서가 반드시 지켜져야 하는 스텝을 구분

## Phase 2 — 의존성 지도

스텝 간 의존 관계를 파악한다.

- "A가 끝나야 B를 시작할 수 있다" 는 관계를 명시
- 병목이 될 가능성이 높은 스텝을 식별

## Phase 3 — 실수 위험 지점

**내가 똑같은 실수를 반복할 가능성이 높은 순간**을 미리 식별한다:

| 유형 | 정의 |
|------|------|
| **낙관 편향** | 잘 될 거라 생각하고 검증을 건너뛰는 순간 |
| **관성 복귀** | 새 구조를 실행하다가 익숙한 옛 방식으로 되돌아가는 순간 |
| **완료 착각** | 아직 끝나지 않았는데 끝났다고 착각하는 순간 |
| **우선순위 역전** | 중요한 일 대신 급하고 쉬운 일을 먼저 하는 순간 |
| **정보 과소** | 판단에 필요한 정보가 부족한데 그냥 진행하는 순간 |

## Phase 4 — 방어 장치 설계

각 위험 지점마다:

- **트리거**: 어떤 상황이 오면 위험한가
- **조기 신호**: 내가 실수하려는 걸 알아챌 수 있는 징후
- **차단 행동**: 그 순간 해야 할 구체적 행동
