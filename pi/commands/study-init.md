# 학습 프로젝트 초기화

학습 주제: **${1:?(주제를 지정하세요)}**

현재 cwd(세션 기준 디렉토리) 아래에 새 학습 프로젝트를 세팅한다.
표준 레퍼런스: 이미 생성된 `study-mysql8/` 프로젝트의 `README.md`, `AGENTS.md` —
같은 구조와 톤을 유지하고, 내용은 주제에 맞게 구체화한다.

## 1. 디렉토리
`study-{slug}` 생성. slug 규칙:
- 소문자, 띄어쓰기→하이픈, 버전 숫자는 유지
- "MySQL 8" → `study-mysql8`
- "Kubernetes" → `study-kubernetes`
- "Go 언어" → `study-go`

## 2. README.md
해당 기술 분야의 커리큘럼을 **기초(0~1년차)부터 8년차 시니어 역량까지** 설계:

- **실습 환경 세팅**: 로컬 샌드박스 중심. 운영/공유 환경은 절대 연습장으로 쓰지 않는다는 원칙 명시. 이 문서에서 샌드박스 실행 명령어는 cwd 기준으로 작성하지 말고 범용적으로 기술한다.
  - 서버 구동이 필요한 학습(Docker 등)은 호스트 포트 충돌을 방지하기 위해 기본 포트(3306, 6379, 5432, 8080 등) 대신 다른 포트(3307, 6380, 5433, 8081 등)를 사용한다.
- **Phase 1 기초(0~1년차)** — "기본기를 다룬다": 핵심 문법/개념, 기본 조작, 데이터/입출력
- **Phase 2 중급(1~3년차)** — "왜 이렇게 동작하는가": 내부 구조, 성능 기초, 설계 원리
- **Phase 3 고급(3~5년차)** — "대규모에서도 작동하게": 튜닝, 동시성, 확장성, 안정성
- **Phase 4 엔지니어링(5~8년차)** — "시스템을 운영하고 설계한다": 관측성, 장애 대응, 아키텍처, 운영/마이그레이션
- 각 Phase: 하위 주제(3~7개) + **체크포인트**(통과 기준, 동사형)
- **추천 학습 리소스**: 책/공식문서/블로그/실습 데이터셋 표
- **학습 원칙 요약**: 5개 내외. 직접 실습·근거(지표/로그/결과) 중심·샌드박스 강조

> 표준 레퍼런스(`study-mysql8/`)의 Phase 구조(목표→하위주제→체크포인트)와 학습 원칙 5개를 뼈대로 쓰되,
> 각 항목은 주제 도메인에 맞는 실제 개념으로 채운다. 빈칸 복사 금지.

## 3. 챕터 디렉토리 규칙
각 Phase의 하위 학습 단위는 챕터 디렉토리로 분리한다.

```
study-{slug}/
├── README.md
├── AGENTS.md
├── SETUP.md
├── ch-01-{영문-슬러그}/
│   ├── README.md           # 챕터 개요 + 학습 목표
│   ├── diagnosis.md        # 사전평가 결과 + 난이도 + 약점 기록
│   ├── test.md             # 학습 후 테스트 문제
│   ├── lab/                # 실습 산출물 (확장자 도메인 자유)
│   └── review/
│       ├── blank-recall.md # 백지 회상: 5개 핵심 아이디어 vs 학습자 답 + STRONG/WEAK/WRONG 분급
│       ├── gap-fill.md     # WEAK/WRONG 보충 (다른 비유로 정정, recall gap만)
│       ├── self-lecture.md # 셀프렉처: 에이전트가 호기심 많은 학생 역할
│       ├── analogy-lock.md # 비유 잠금: 비유 2개 + 깨지는 지점 + 1문장 요약
│       ├── schedule.md     # 주기적 반복 일정 기록
│       └── learning-gaps.md# ★ 본 학습 누락 (복습 아님, chapter 회귀 신호)
└── ch-02-{영문-슬러그}/
```

규칙:
- 디렉토리명: `ch-{NN}-{영문-슬러그}` (NN은 01부터 시작하는 2자리 숫자)
- 챕터 내 파일명은 **역할 기반**으로 고정(README/diagnosis/quiz/lab/review). 확장자는 학습 주제에 맞춰 자유롭게 둔다.
- `lab/` 산출물 확장자는 도메인이 정한다: SQL은 `.sql`, Go는 `.go`, K8s는 `.yaml` 등. 한 도메인도 여러 확장자가 섞일 수 있다.
- `ch-` 프리픽스는 챕터 구분과 정렬을 위함. Phase 구분은 README.md의 Phase 번호로 한다.

## 4. 챕터 학습 플로우
모든 챕터는 아래 순서로 진행한다:

1. **사전 평가** — `diagnosis.md`에 객관식(80%)/주관식(15%)/서술형(5%) 문제를 풀고 결과를 기록한다.
2. **결과 기록** — `diagnosis.md`에 사전평가 결과 + 난이도 + 약점을 기록. 이후 단계에서 참조한다.
3. **개념 학습** — 사전평가 결과에 맞춰 깊이와 난이도를 조정한다. 이미 아는 것은 가볍게, 약점은 깊이.
4. **실습 수행** — `lab/`에서 직접 실행. SQL이면 실행계획, 코드면 결과/로그를 첨부한다.
5. **테스트** — 통과 기준 미달 시 해당 개념만 재학습. 전체 반복 금지.
6. **복습(`/study-review` 커맨드로 진행)** — 에이전트는 5가지 역할을 번갈아 한다:
   1. `blank-recall.md` (검증자): 원본에서 5개 핵심 아이디어 추출, 학습자 답과 대조해 STRONG/WEAK/WRONG 분급.
   2. `gap-fill.md` (보강자): WEAK/WRONG만 다른 비유로 정정 + 연습문제, 끝에 "가장 먼저 다시 볼 1개" 지정. 학습 누락은 제외.
   3. `self-lecture.md` (호기심 많은 학생): 학습자가 12세 말투로 설명, 에이전트가 "왜요?" 위성 질문으로 빈틈 강제 드러냄.
   4. `analogy-lock.md` (정착자): 빈틈 메운 뒤 비유 2개 + 깨지는 지점 + 1문장 요약으로 고정.
   5. `schedule.md` (스케줄러): 3일/1주/2주 간격, 매번 새 질문 생성.
   - **스코프 가드**: 피드백은 본 학습 범위(concept/lab)로 한정. 벗어나면 `learning-gaps.md`에 분류하고 "본 학습으로 회귀" 안내.

## 5. AGENTS.md
해당 분야의 **시니어 멘토 정체성**을 정의 (MySQL 예시와 같은 8개 섹션):

1. **정체성**: 현업 엔지니어 출신, 새벽 페이지까지 겪은 사람. 학습자를 '예비 동료'로.
2. **목표**: 병목이 아닌 자산을 만드는 엔지니어로 키운다.
3. **교육 원칙**: 손 먼저/이론 나중 · 항상 근거(실행결과·지표·로그) · 근본 원인(증상 아님) · 최소 코드 · 장애 직접 재현 · 안전 타협 없음
4. **음성/톤**: 한국어 직설, 코드 리뷰 방식, 칭찬보다 피드백, 에세이 금지, 모르면 모른다고
5. **하는 것 / 하지 않는 것**
6. **학습자에 대한 기대**: 직접 칠 것·근거로 말할 것·틀릴 용기·구체적 질문
7. **세션 운영 방식**: 진도확인→목표→실습→검토→체크포인트→다음
8. **한 줄 요약**

> 표준 레퍼런스(`study-mysql8/AGENTS.md`)의 문장을 도메인에 맞게 치환. "DB"를 해당 분야 용어로.
> 교육 원칙 6개와 톤은 분야 무관하게 동일하게 유지.
