# ladder-roadmap

"무엇을 어떤 순서로 배워야 하는가"를 설계합니다.
목표를 먼저 정하고, 현재 위치에서 그 목표까지 **가장 효율적인 경로**를 제시합니다.

## 사용 방법

```
/ladder-roadmap $GOAL [$CURRENT_LEVEL]
```

- `$GOAL`: 도달하고 싶은 목표. 예: "백엔드 개발자", "JVM 내부 이해", "네트워크 스택 이해"
- `$CURRENT_LEVEL`: 현재 수준 (선택). 생략 시 질문으로 파악.

---

## 설계 원칙

1. **목표 구체화 먼저**: "백엔드 개발자"처럼 넓은 목표는 먼저 범위를 좁힌다.
   - "어떤 수준의 백엔드 개발자인가? 취직? 시스템 설계? 오픈소스 기여?"

2. **현재 수준 진단**: 목표와 연관된 핵심 선행 개념 3~5개에 대해 자가 평가를 받는다.
   ```
   [개념 X]를 설명할 수 있다: ✅ / ⚠️ / ❌
   ```

3. **경로 출력 형식**:
   ```
   ## 학습 로드맵: [목표]

   현재 위치: [수준 요약]
   목표까지 예상 소요: [N주 기준 X시간/주]

   ### Phase 1: 기초 확보 (필수)
   1. [주제] — 이유: [왜 이게 먼저인가]
      → /ladder-explain [주제]
   2. [주제] — 이유: ...

   ### Phase 2: 핵심 구축
   3. [주제]
   4. [주제]

   ### Phase 3: 심화 / 실전
   5. [주제]
   6. [주제]

   ### 체크포인트
   각 Phase 끝에서 /ladder-quiz-me 로 점검 권장.
   ```

4. **유연성 명시**: 경로는 제안일 뿐이며, 흥미와 필요에 따라 순서를 바꿔도 된다.
