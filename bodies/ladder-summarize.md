# ladder-summarize

`/ladder-explain`으로 깊이 이해한 내용을 **기억에 남는 최소 단위**로 압축합니다.
"다음 주에 누군가에게 설명해야 한다면 뭘 기억해야 하는가"가 기준입니다.

## 사용 방법

```
/ladder-summarize $TOPIC
```

- `$TOPIC`: 압축할 주제. 생략 시 직전 대화 맥락에서 자동 추출.

---

## 출력 구조

### 1. 한 줄 비유
> 이 개념 전체를 일상 비유 한 문장으로 압축.
> 예: "TCP는 전화 통화다 — 연결 확인하고, 말하고, 끊었다고 인사하는."

### 2. 핵심 인과 체인 (3~5단계)
```
A → B → C → D
(각 화살표에 "왜?"를 한 단어로 표현)
```

### 3. 층위별 핵심 한 줄
각 관련 층위에서 기억해야 할 것 딱 한 줄씩:
- [층N]: ...

### 4. 자주 헷갈리는 것 (있을 경우)
```
❌ 많이 오해하는 것: "..."
✅ 실제: "..."
```

### 5. 30초 설명 스크립트
누군가에게 이 개념을 30초 안에 설명해야 할 때 쓸 수 있는 텍스트.

---

## 원칙

- 상세한 것을 다 담으려 하지 않는다. **잊지 않을 최소한**만.
- 비유는 1개. 다수의 비유는 오히려 혼란을 유발한다.
- 인과 체인은 5단계 이하. 넘어가면 중간 단계를 묶는다.
