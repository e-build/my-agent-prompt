# ladder-show-code

개념 설명에서 끝나지 않고, 실제로 어떤 코드가 되는지 보여줍니다.
각 줄마다 "이 줄이 어느 층위의 어떤 동작인지"를 명시합니다.

## 사용 방법

```
/ladder-show-code $TOPIC [$LANGUAGE]
```

- `$TOPIC`: 코드로 보고 싶은 개념. 예: "TCP 3-way handshake", "GC 트리거 조건"
- `$LANGUAGE`: 언어 지정 (선택). 생략 시 가장 직관적인 언어로 선택.

---

## 출력 원칙

1. **코드만 던지는 것 금지**: 모든 코드는 줄별 설명을 동반한다.

2. **층위 태그 명시**: 각 줄/블록에 어느 층위와 연관된 동작인지 태그를 붙인다.
   ```
   // [층4: 언어/컴파일러] 힙에 객체 할당 요청
   const obj = new SomeClass();

   // [층3: OS/런타임] 실제 메모리 페이지를 OS에 요청
   // [층2: 하드웨어] 물리 메모리 주소로 매핑
   ```

3. **실패 시나리오 우선**: 정상 작동 코드 전에 "이 코드가 없으면 뭐가 터지나"를 먼저 보여준다.

4. **점진적 공개**:
   - Step 1: 가장 단순한 형태 (작동만 함)
   - Step 2: 실제 프로덕션에 가까운 형태 (예외 처리, 성능 고려)
   - Step 3: 내부 동작 들여다보기 (언어/런타임이 실제로 하는 일)

5. **강화 옵션** (사용자가 추가 지정 가능):
   - `+실패` → 각 줄에서 발생 가능한 예외/에러 케이스 추가
   - `+성능` → 시간복잡도, 메모리 사용량 주석 추가
   - `+비교` → 다른 언어/라이브러리로 같은 것을 구현했을 때와 비교
