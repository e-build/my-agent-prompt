# ladder-find-gaps

"어디서부터 시작해야 할지 모르겠다"는 상황을 해결합니다.
현재 알고 있는 것을 진단하고, 목표 주제를 이해하기 위해 **무엇을 먼저 채워야 하는지** 찾아줍니다.

## 사용 방법

```
/ladder-find-gaps $TOPIC
```

- `$TOPIC`: 이해하고 싶은 목표 주제. 예: "JVM GC", "HTTPS TLS handshake"

---

## 진단 방식

### Phase 1: 목표 주제 분해
목표 주제를 이해하기 위해 필요한 선행 개념 트리를 작성한다.
```
목표: [주제]
  ├── 필요 개념 A
  │     ├── 필요 개념 A-1
  │     └── 필요 개념 A-2
  ├── 필요 개념 B
  └── 필요 개념 C
        └── 필요 개념 C-1
```

### Phase 2: 현재 수준 파악
각 선행 개념에 대해 사용자에게 **단답 질문**을 던진다 (한 번에 하나씩):
```
[선행 개념 A]에 대해 한 문장으로 설명해보세요.
(모르면 "모른다"고 해도 됩니다.)
```

### Phase 3: 갭 지도 출력
```
## 현재 상태

✅ 알고 있음: A, C-1
⚠️  불완전함: A-1 (방향이 맞지만 인과가 빠짐)
❌ 모름: A-2, B, C

## 권장 학습 순서
1. B (기초. 없으면 A-2도 이해 안 됨)
2. A-2 (B를 알면 빠르게 이해 가능)
3. C (독립적)
4. 목표 주제 진입 가능
```
