# ladder-quiz-me

`/ladder-explain`으로 설명을 들은 후, 진짜 이해했는지 확인합니다.
단순 암기가 아닌 **인과 추론**과 **반사실적 사고**를 테스트합니다.

## 사용 방법

```
/ladder-quiz-me $TOPIC
```

- `$TOPIC`: 테스트할 주제. 생략 시 직전 `ladder-explain` 주제로 자동 적용.

---

## 출제 원칙

1. **층위별 출제**: 6개 층 각각에서 최소 1문제. 전체를 얕게 덮는 게 아니라 핵심 연결 고리를 찌른다.

2. **문제 유형 3가지**:
   - **인과 질문** — "A가 B를 일으키는 이유는?"
   - **반사실 질문** — "만약 X가 없었다면 어떻게 됐을까?"
   - **비유 검증** — "앞서 든 비유에서 실제와 다른 점은?"

3. **단계별 진행**: 한 번에 1문제씩. 답을 받은 뒤 다음 문제로 넘어간다.

4. **채점 기준**:
   - 정확한 용어보다 **인과 관계가 맞는지**를 본다.
   - "맞다/틀리다"보다 "어디서 체인이 끊겼는가"를 짚어준다.

5. **피드백 형식**:
   ```
   ✅ 맞음: [어떤 부분이 정확한지]
   ⚠️  아쉬움: [어디서 인과가 끊겼는지]
   💡 보완: [놓친 연결 고리 한 줄 설명]
   ```

---

## 출력 구조

```
## 문제 N/N — [층위 이름]

[문제 내용]

> 답을 입력해 주세요. 모르면 "모르겠어"라고 해도 됩니다.
```
